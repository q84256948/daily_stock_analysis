# A 股深度投研报告分析逻辑优化方案

本文审计并优化 A 股深度投研报告的分析逻辑。目标是把报告从传统“五层均衡 + 技术节奏”调整为“消息面与产业链主导，国产替代 / 材料独特地位 / 中美平行链重点分析，技术面降为辅助择时”的框架。

本方案只规划实现，不写代码。

## 目标

- 深度投研主判断由消息面、产业链位置、国产替代、材料或工艺独特地位、中美产业链关系驱动。
- 技术面只作为次要节奏参考，不再主导评级和核心投资结论。
- 信息源覆盖官方公告、公开新闻、行业资料、国内社区、海外社区等公开来源。
- 国产替代和中美平行链必须先判断适用性；白酒、银行、保险、传统地产等低相关行业不硬套。
- 保留现有报告生成、存储、PDF、SSE API 主链路，优先通过 prompt、工具集和质量校验收敛。

## 现状审计

当前深度投研 executor 在 `src/agent/deep_research_executor.py`，system prompt 从 `data/deep_research/system_prompt.md` 加载。

现有框架是“五层穿透”：

```text
宏观 → 产业 → 财务 → 估值 → 博弈
```

当前强制工具检查：

- 宏观：`get_market_indices` + `search_comprehensive_intel`
- 产业：`get_sector_rankings` + `search_comprehensive_intel`
- 财务：`get_stock_info`
- 估值：内容层面检查
- 博弈：`analyze_trend` / `get_chip_distribution` / `get_capital_flow`

主要问题：

- 宏观层容易泛化，占用报告篇幅，但对单只 A 股长期产业链判断贡献有限。
- 技术面位于“博弈层”并参与强制校验，容易让最终报告偏短线节奏。
- 产业链章节已有，但缺少对国产替代、材料/工艺独特地位、中美平行链的强制展开。
- `search_comprehensive_intel` 能覆盖部分新闻与行业动态，但没有明确区分官方、新闻、社区、海外社区和推断。
- 供应链双源校验工具 `verify_supply_chain_evidence` 已存在，但当前主要注册给供应链报告，不是深度投研核心路径。

## 优化后分析框架

报告一级结构建议调整为：

```text
# {股票名称}（{股票代码}）深度投研报告

## 投资结论
## 一、消息面与产业政策
## 二、产业链位置与瓶颈环节
## 三、国产替代与平行替换
## 四、材料 / 工艺 / 环节独特地位
## 五、中美产业链关系
## 六、财务与估值验证
## 七、技术面与筹码节奏（次要）
## 八、风险、证伪条件与下一步验证
```

### 投资结论

必须先回答：

- 这家公司是否具备值得研究的产业链位置。
- 核心逻辑来自消息面变化、产业链瓶颈、国产替代、中美链重构，还是纯财务/估值。
- 国产替代和中美平行链是否适用；不适用时直接说明。
- 技术面是否支持当前节奏，但不得覆盖产业链主判断。

### 一、消息面与产业政策

宏观与政策环境压缩到本章，不再单独成章。

必含：

- 官方公告与财报线索。
- 公开新闻与产业事件。
- 国内外社区讨论与市场分歧。
- 政策敏感度与监管风险。
- 消息面可信度分层。

社区信息只用于识别关注度、争议点、传闻线索和市场认知差，不得单独作为事实确认。

### 二、产业链位置与瓶颈环节

必含：

- 上游 / 中游 / 下游图谱。
- 公司所在环节。
- 是否属于卡点、瓶颈、关键材料、关键设备、关键工艺、关键客户入口。
- 东方财富 + 同花顺双源校验状态。

建议把 `verify_supply_chain_evidence` 加入深度投研工具集，用于 A 股公司 / 板块归属核验。

### 三、国产替代与平行替换

先判断适用性。

强相关行业：

- 半导体、AI 算力、光模块、CPO、芯片设备材料。
- 新能源车、锂电、光伏、储能。
- 高端制造、机器人、军工、工业母机。
- 医药器械、创新药关键设备或材料。
- 工业软件、信创、基础材料、关键零部件。

低相关行业：

- 白酒、银行、保险、传统地产、公用事业、纯消费品牌等。

低相关行业固定写法：

```text
国产替代 / 中美平行链：低相关。本公司核心逻辑不来自进口替代或中美链重构，报告不强行套用。
```

强相关行业必须区分替代阶段：

- `传闻`
- `送样`
- `验证`
- `导入`
- `量产`

不得把社区传闻或媒体标题写成“已替代”。

### 四、材料 / 工艺 / 环节独特地位

重点看：

- 材料或工艺是否独特。
- 是否存在专利、配方、良率、纯度、设备、认证壁垒。
- 客户验证周期和切换成本。
- 扩产难度和边际产能约束。
- 国内同行是否可以复制，多久复制。

本章必须回答“独特地位是真壁垒，还是短期题材标签”。

### 五、中美产业链关系

先判断适用性。

强相关时必须展开：

- 公司在中国链中的位置。
- 公司在美国 / 全球链中的位置。
- 出口管制、制裁、设备材料限制。
- 是否受益于平行替换。
- 是否受损于海外客户、设备、EDA、材料、IP、认证受限。

低相关行业不硬写中美链。

### 六、财务与估值验证

财务和估值用于验证产业链逻辑是否兑现，而不是孤立给倍数。

必含：

- 收入、利润、毛利率、ROE、现金流。
- 产业链逻辑是否已反映在收入或毛利率中。
- 估值是否已经透支国产替代 / 中美链预期。
- 三情景估值。

### 七、技术面与筹码节奏（次要）

技术面只回答：

- 当前价格位置是否适合建仓 / 等待。
- 关键支撑阻力。
- 资金流和筹码是否与产业逻辑共振。

强制约束：

- 技术面不得推翻产业链主判断。
- 技术面只影响节奏，不影响公司是否具备长期研究价值。
- 技术面章节篇幅应明显低于产业链和消息面章节。

### 八、风险、证伪条件与下一步验证

必须列出：

- 能证伪产业链地位的事实。
- 能证伪国产替代进度的事实。
- 能证伪中美平行链受益的事实。
- 下一步需要查的公告、客户、订单、产能、认证、监管信息。

## 信息源策略

每份报告的信息源分层：

| 来源等级 | 来源类型 | 用途 |
| --- | --- | --- |
| `primary` | 公告、财报、交易所、巨潮、互动易、监管、公司官网 | 可支撑事实确认 |
| `news` | 财经新闻、行业媒体、地方产业新闻、公开新闻 | 可支撑事件和市场变化 |
| `industry` | 行业协会、产业报告、专业研究机构 | 可支撑产业趋势和格局 |
| `community_cn` | 雪球、东方财富股吧、同花顺社区、微博、知乎、贴吧 | 识别市场分歧、传闻和关注度 |
| `community_global` | Reddit、X、海外论坛、海外投资讨论区 | 识别海外认知和争议 |
| `inferred` | 基于公开信息推断 | 必须写明推断依据 |
| `unverified` | 未找到可靠证据 | 不得写成确认 |

规则：

- 社区信息不得单独支撑“确认”“实锤”“已导入”“已量产”。
- 国产替代进度必须有阶段标签。
- 中美链判断必须说明替代谁、在哪个环节、受什么限制、验证周期多长。
- 所有具体客户、订单、份额、产能、价格、毛利率必须标明来源等级。

## 工具调整建议

### P0：仅改 prompt 和校验

优先修改：

- `data/deep_research/system_prompt.md`
- `src/agent/deep_research_executor.py::_build_user_message`
- `src/agent/deep_research_validator.py`

目标：

- 报告骨架改为新八章。
- 用户消息中的强制要求改为“消息面 + 产业链 + 国产替代 + 中美链 + 财务估值 + 技术辅助”。
- validator 从旧五层校验改为新层级校验。

### P1：接入供应链双源校验

把 `verify_supply_chain_evidence` 加入深度投研工具集。

深度投研产业链章节要求：

- A 股板块 / 概念 / 公司归属判断必须调用双源校验。
- 未得到 `confirmed` 时只能写单源支持、冲突或待核验。

### P2：补社区与海外讨论检索

新增或扩展工具：

```text
search_market_discussion
```

覆盖：

- 国内社区：雪球、东方财富股吧、同花顺社区、微博、知乎、贴吧。
- 海外社区：Reddit、X、海外论坛、海外投资讨论区。
- 返回结构包含 `source_type`、`source_name`、`title`、`snippet`、`url`、`published_date`、`claim`、`reliability_hint`。

如果不新增工具，也可先扩展 `search_comprehensive_intel` 的查询维度，但独立工具更容易测试和约束。

### P3：信息源持久化与报告引用

社区源和公开新闻最好复用现有 news intel persistence，但必须带维度：

- `official`
- `news`
- `industry`
- `community_cn`
- `community_global`

## 质量校验建议

新的 validator 层级：

| 层级 | 内容关键词 | 工具要求 |
| --- | --- | --- |
| 消息面与产业政策 | 消息、公告、新闻、社区、政策、监管、分歧 | `search_stock_news` 或 `search_comprehensive_intel` |
| 产业链位置 | 产业链、上游、中游、下游、瓶颈、卡点、板块归属 | `get_sector_rankings`，P1 后加 `verify_supply_chain_evidence` |
| 国产替代 | 国产替代、进口替代、平行替换、送样、验证、导入、量产、不适用 | 内容必含适用性判断 |
| 材料/工艺独特地位 | 材料、工艺、良率、纯度、认证、扩产、客户验证 | 内容必含 |
| 中美产业链关系 | 中美、美国链、中国链、出口管制、制裁、平行链、不适用 | 内容必含适用性判断 |
| 财务与估值 | 营收、利润、毛利率、ROE、估值、PE、PB、目标价 | `get_stock_info` |
| 技术与筹码 | 技术面、筹码、资金流、支撑、阻力、趋势 | `analyze_trend` / `get_chip_distribution` / `get_capital_flow` 满足其一 |

技术与筹码层可降低分值权重。缺技术面不应让报告失败，但应标为缺少择时信息。

## Prompt 输出约束

建议在 system prompt 中新增硬约束：

- 报告结论必须先给产业链和消息面判断，再给技术面。
- 技术面章节不得超过报告主体的 15%。
- 社区源必须单独列入“市场分歧”，不得混入官方事实。
- 国产替代和中美链先判断适用性；低相关行业必须写低相关或不适用。
- 白酒、银行、保险、传统地产、公用事业等低相关行业不强行套国产替代。
- 不能用“国产替代受益”作为万能结论，必须说明替代对象、替代环节、替代阶段和证据等级。

## 实施步骤

1. 更新 `data/deep_research/system_prompt.md`
   - 替换旧报告骨架。
   - 压缩宏观到“消息面与产业政策”。
   - 明确技术面降权。

2. 更新 `DeepResearchExecutor._build_user_message`
   - 调整强制工具说明。
   - 明确数据收集顺序：消息面 → 产业链 → 国产替代 / 中美链 → 财务估值 → 技术辅助。

3. 更新 `DeepResearchValidator`
   - 替换 `_LAYER_REQUIREMENTS`。
   - 更新 `_REQUIRED_SECTIONS`。
   - 技术面缺失只降分，不直接触发重生成失败。

4. 更新 `build_deep_research_executor`
   - P1 时加入 `verify_supply_chain_evidence`。
   - P2 时加入 `search_market_discussion`。

5. 补测试和文档。

## 测试计划

后端：

- system prompt 包含新八章标题。
- user message 不再要求旧五层穿透。
- validator 对新结构报告通过。
- validator 对缺国产替代适用性判断的报告降分。
- validator 对白酒/银行类“不适用”写法不误判失败。
- validator 对技术面缺失只标记择时信息不足，不阻断主报告。
- deep research 工具集 P1 后包含 `verify_supply_chain_evidence`。

工具：

- 社区检索工具返回 source_type。
- 国内外社区源单源失败不拖垮整体。
- 社区结果不能被标为 primary。

报告 e2e mock：

- 生成报告结论先写消息面 / 产业链，不以技术面开头。
- 产业链章节含双源校验状态。
- 国产替代章节含适用性判断。
- 中美链章节含适用性判断。
- 白酒 / 银行 fixture 报告写低相关，不硬套国产替代。

## 风险与降级

- 海外社区和 X 搜索可能受 provider 限制，失败时只标“海外社区未检索到 / 搜索不可用”，不影响主报告。
- 社区噪音高，必须限制为市场分歧和线索，不参与事实确认。
- 新框架会改变历史报告风格，但不改变 API / storage / PDF 契约。
- 若 validator 过严，会增加重生成成本；技术面应降权，不宜作为失败主因。

## 回滚方式

- 回退 `data/deep_research/system_prompt.md`。
- 回退 `DeepResearchExecutor._build_user_message`。
- 回退 `DeepResearchValidator` 层级要求。
- 从深度投研工具集移除新增工具。
- storage、API、Web、PDF 不变，无数据迁移。

## 最终结论

优化后的方案保留现有深度投研主链路，只调整分析重心和质量校验。第一阶段应先改 prompt 与 validator，把报告结论拉回消息面、产业链、国产替代、材料独特地位和中美平行链；技术面只作为择时辅助。第二阶段再补供应链双源校验和国内外社区检索工具。
